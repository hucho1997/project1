# 프로젝트 지침: 포켓몬 로컬 DB

## 1) 프로젝트 개요

- 목적: 포켓몬 본가 시리즈 데이터를 로컬 DB로 구축 (개인 취미)
- 데이터 소스: PokeAPI CSV 원본 데이터

## 2) 절대 금지 사항

### 이미지 사용 금지

- 이 프로젝트에서 이미지 파일은 **어떠한 경우에도** 사용하지 않는다.
- 금지 대상:
  - 포켓몬 스프라이트, 아이콘, 일러스트 등 모든 형태의 이미지 파일 (png, jpg, gif, svg, webp 등)
  - 외부 이미지 URL 참조 및 핫링크
  - Base64 인코딩된 이미지 데이터
  - 이미지를 포함하는 외부 라이브러리/패키지 의존성
- 데이터는 텍스트/수치 정보만으로 구성한다.
- AI는 이미지 관련 제안, 다운로드, 참조를 일절 하지 않는다.

## 3) AI 역할 및 권한

### 프로젝트 구조 관리 위임

- 폴더 구조, 파일 명명, 데이터 설계는 **AI가 판단하여 진행**한다.
- 사용자는 프로젝트 방향과 기능 요구사항을 결정하고, 기술적 구현 구조는 AI에게 위임한다.
- AI는 구조 변경(폴더 생성, 파일 재배치 등) 시 **사전에 변경 내용을 설명하고 합의 후 실행**한다.
- 단, 기존 파일의 삭제·이동은 반드시 사용자 확인을 받는다.

### 설명 방식

- 사용자가 기술적 개념을 질문할 때, AI는 **교수가 학생에게 설명하듯** 응답한다.
  - 틀린 내용을 수용하지 않고, 왜 다른지 근거를 들어 설명한다.
  - 여러 선택지가 있을 때 각각의 장단점을 설명하고 사용자가 판단하도록 한다.
  - 단, 말투는 평소대로 유지한다 (교수 말투 X).

## 4) 프로젝트 구조

```
01/
├── 00_PROJECT_INSTRUCTIONS.md
├── data/
│   └── csv/                    ← PokeAPI 원본 CSV (가공 전 raw 데이터)
├── db/
│   └── pokemon.db              ← SQLite 로컬 데이터베이스 (가공 후)
└── scripts/
    └── build_db.py             ← CSV → SQLite 변환 스크립트
```

- 실무 표준 구조: 원본 데이터(data/)와 가공 산출물(db/)을 동등하게 분리한다.
- 원본 CSV는 여러 산출물의 공통 소스가 될 수 있으므로 독립 배치한다.
- 구조 변경이 필요할 때 AI가 사전 설명 후 진행한다.
